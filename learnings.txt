# orb-browser learnings

## The /task "freeze" — it's checkpointing, not a bug (2026-04-01)

### What we thought
Python HTTP libraries were blocking the asyncio event loop, freezing the server.

### What's actually happening
Orb automatically checkpoints (CRIU freezes) the VM when it detects the process
is waiting on an outbound API call (e.g., LLM request via OpenRouter). The entire
VM is suspended to NVMe — health, task polling, everything stops responding.

When a new inbound request arrives, Orb promotes (restores) the VM. This takes
~1-2 seconds. Our curl timeouts of 5s were sometimes too short.

### Evidence
- admin.orbcloud.dev showed VM 83355acb as FROZEN while /task was running
- Process state showed FROZEN in the processes table
- Health responded fine with a 30s timeout (took 1.3s to wake)

### What this means
1. The original ainvoke code was fine — no event loop blocking
2. The subprocess curl workaround is unnecessary (but doesn't hurt)
3. Background task + polling is still good architecture (returns task_id immediately)
4. Polling clients need longer timeouts (30s+) to account for checkpoint wake time

### Orb's LLM Proxy
The admin dashboard shows an "LLM Proxy" with 113 requests tracked. Orb has a
built-in LLM proxy — using it might avoid triggering checkpoints during LLM calls.
Worth investigating.

### Model compatibility (OpenRouter)
- `anthropic/claude-3.5-haiku` does NOT support image input via OpenRouter (Bedrock)
- `google/gemini-2.0-flash-001` works for vision tasks via OpenRouter
- `qwen/qwen-vl-plus` works for vision, cheapest option
- Model name format: `anthropic/claude-3.5-haiku` (with dot), NOT `anthropic/claude-3-5-haiku-20241022`

### Architecture
- /task returns task_id immediately, runs vision loop as background asyncio task
- GET /task/{id} for polling status/results
- start_url parameter pre-navigates before vision loop starts
- Message history trimmed to last 3 screenshots to avoid payload bloat
- LLM calls use subprocess curl (could revert to ainvoke now we understand the freeze)
