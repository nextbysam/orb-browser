# orb-browser learnings

## Vision agent /task — working architecture (2026-04-02)

### How it works
- `/task` is SYNCHRONOUS — HTTP request stays open
- Agent calls `https://openrouter.ai/api/v1/chat/completions` directly
- Orb intercepts outbound calls to openrouter.ai (it's in LLM_DOMAINS)
- Orb checkpoints VM during LLM call, restores when response arrives
- Client uses long timeout (5-10 min)

### What works
- Text-heavy pages (HN, uniclaw): 3-9s, works perfectly
- Vision with Gemini Flash via OpenRouter
- Retry logic catches httpx.ReadTimeout from checkpoint/restore

### Open issue: large image payloads timeout via Orb proxy
- Pages with images (CERN 114KB, Cloudflare, GitHub) timeout at 300s+
- Same OpenRouter call returns in 4s when called locally (not through Orb)
- Small payloads (6-9KB) work fine through proxy
- Proxy stats show 0 errors but 244s avg latency
- Theory: proxy buffer + checkpoint cycle breaks for large payloads
- NEEDS ORB-SIDE FIX — not a code issue on our end

### Model compatibility (OpenRouter)
- `anthropic/claude-3.5-haiku` does NOT support image input (Bedrock)
- `google/gemini-2.0-flash-001` works for vision, good quality
- `qwen/qwen-vl-plus` works for vision, cheapest
- Dot notation: `anthropic/claude-3.5-haiku` not `anthropic/claude-3-5-haiku-20241022`

### Orb proxy internals
- openrouter.ai is in LLM_DOMAINS — intercepted at network level
- OPENAI_BASE_URL is set but only for /v1/messages (Anthropic format)
- admin.orbcloud.dev shows VM status, checkpoint stats, LLM proxy metrics
- Checkpoint: 100% success, avg 757ms, avg size 45.2 MB
- Restore: 100% success, avg wake 507ms, p95 768ms

### What did NOT work
- Background task + polling: VM doesn't auto-wake for poll requests
- OPENAI_BASE_URL proxy: only handles Anthropic /v1/messages, not /v1/chat/completions
- Short timeouts: need 300s+ for httpx, 600s for client
- Python HTTP libs (httpx, urllib) DON'T block the event loop — the "freeze" was always Orb checkpointing
